# AGENTS.md — frontend

Rules for the React + TypeScript UI. The root `AGENTS.md` still applies.

## Never write `TODO` or `FIXME`

Repeated here because this is where it bites. `.eslintrc.json` sets `no-warning-comments`
to `error` for those terms and lint runs with `--max-warnings 0`, so one marker fails CI.

## Layout

| What | Where | Naming |
| --- | --- | --- |
| Redux state | `src/features/` | `<name>Slice.ts`, `<name>Selectors.ts`, `<name>Thunks.ts` |
| API URLs | `src/api/apiEndpoints.ts` | grouped `<resource>Endpoints` objects |
| API calls | `src/api/api-functions/<resource>.ts` | one function per endpoint |
| Types | `src/types/` | `Media.ts`, `API.ts`, … |
| Hooks | `src/hooks/` | `use<Thing>.ts` |
| Components | `src/components/<Area>/` | `PascalCase.tsx` |
| Tests | `__tests__/` beside the code | `<Name>.test.tsx` |

`@/` resolves to `frontend/src` (`tsconfig.json`). Use it instead of deep relative paths.

## State

Redux Toolkit `createSlice`. Keep reducers small and total — validate indices and ranges
inside the reducer rather than at the call site (see `setCurrentViewIndex` in
`src/features/videoSlice.ts`). Derived data goes in a `<name>Selectors.ts`, not in
components.

## API layer

Two steps, always both:

1. Add the URL to the right `<resource>Endpoints` object in `src/api/apiEndpoints.ts`.
   Parameterised URLs are functions that `encodeURIComponent` their arguments.
2. Add a typed wrapper in `src/api/api-functions/<resource>.ts` that uses the shared
   `apiClient` from `../axiosConfig` and returns `Promise<APIResponse>`.

Components never call `axios` directly.

## Components

- `src/components/ui/` is generated by shadcn. Do not hand-edit it; regenerate instead.
- Tailwind v4 with `prettier-plugin-tailwindcss`. Never hand-order class names — let
  Prettier sort them.

## Tests

Jest + React Testing Library (`jest.config.ts`, `jest.setup.ts`). Test behaviour through
the rendered output, not implementation details. Run with `npm test`.

## Before you finish

```bash
npm run lint:check && npm run format:check && npm test
```
